# 任務管理系統 (Task Management System) 路由設計文件

## 1. 路由總覽表格

| 功能 | HTTP 方法 | URL 路徑 | 對應模板 | 說明 |
| :--- | :--- | :--- | :--- | :--- |
| 顯示任務清單 | GET | `/` | `index.html` | 顯示所有或篩選過的任務 |
| 新增任務 | POST | `/add` | — | 接收任務標題並重導向至首頁 |
| 切換完成狀態 | POST | `/toggle/<int:id>` | — | 切換任務完成/未完成狀態後重導向 |
| 刪除任務 | POST | `/delete/<int:id>` | — | 刪除任務後重導向 |

## 2. 每個路由的詳細說明

### `GET /`
- **功能**: 顯示任務清單
- **輸入**: `filter` (Query String, 可選: `done`, `todo`)
- **處理邏輯**: 呼叫 `Task.get_all(filter_status)`
- **輸出**: 渲染 `index.html` 並傳遞任務列表
- **錯誤處理**: 無特殊錯誤

### `POST /add`
- **功能**: 新增任務
- **輸入**: `title` (表單欄位, 必填)
- **處理邏輯**: 驗證標題不為空，呼叫 `Task.create(title)`
- **輸出**: 重導向至 `/`
- **錯誤處理**: 若標題為空，則重導向回首頁或顯示錯誤訊息

### `POST /toggle/<int:id>`
- **功能**: 切換任務完成狀態
- **輸入**: `id` (URL 參數)
- **處理邏輯**: 呼叫 `Task.update_status(id)`
- **輸出**: 重導向至 `/`
- **錯誤處理**: 若 ID 不存在，回傳 404

### `POST /delete/<int:id>`
- **功能**: 刪除任務
- **輸入**: `id` (URL 參數)
- **處理邏輯**: 呼叫 `Task.delete(id)`
- **輸出**: 重導向至 `/`
- **錯誤處理**: 若 ID 不存在，回傳 404

## 3. Jinja2 模板清單

| 模板檔案 | 繼承模板 | 說明 |
| :--- | :--- | :--- |
| `base.html` | 無 | 包含網站共用的 HTML 結構、CSS/JS 連結 |
| `index.html` | `base.html` | 顯示任務清單頁面，包含新增表單與篩選器 |
