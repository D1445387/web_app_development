# 任務管理系統 (Task Management System) PRD

## 1. 專案概述

### 背景與動機
在繁忙的日常生活中，個人用戶常面臨任務繁雜、難以追蹤進度的問題。本專案旨在提供一個簡潔、高效的任務管理工具，協助用戶紀錄並追蹤待辦事項，提升工作與生活效率。

### 目標用戶
需要管理日常工作與私人瑣事的個人用戶。

### 核心價值主張
提供直觀的介面，讓用戶能快速新增、管理及檢視任務進度，減少遺漏重要事項的風險。

---

## 2. 功能需求

以下功能將協助用戶有效管理任務：

- **新增任務**
  - 作為一名用戶，我希望能夠新增一個任務，以便記錄我需要完成的事項。
- **標記任務完成**
  - 作為一名用戶，我希望能夠將任務標記為已完成，以便清楚掌握哪些工作已經達成。
- **刪除任務**
  - 作為一名用戶，我希望可以刪除不再需要的任務，以便保持任務清單的整潔。
- **顯示所有任務清單**
  - 作為一名用戶，我希望看到一個清晰的清單，列出我所有的任務，以便對工作量有全局的認識。
- **依完成狀態篩選**
  - 作為一名用戶，我希望可以根據任務的「已完成」或「未完成」狀態進行篩選，以便專注於當前的待辦事項。

---

## 3. 非功能需求

### 技術限制
- **後端框架**: Flask
- **模板引擎**: Jinja2
- **資料庫**: SQLite
- **前端工具**: Vanilla CSS, JavaScript

### 效能與安全考量
- 系統應能在秒級內響應任務的增刪改查操作。
- 資料庫應確保任務數據的持久性，重啟後不遺失資料。
- 介面需具備基本的輸入驗證，防止空任務被新增。

---

## 4. MVP 範圍

本專案將優先開發以下功能：

| 功能層級 | 功能描述 |
| :--- | :--- |
| **Must Have** | 新增任務、顯示任務清單、標記完成/未完成 |
| **Should Have** | 刪除任務、依狀態篩選任務 |
| **Nice to Have** | 任務優先級設定、任務截止日期、搜尋功能 |
